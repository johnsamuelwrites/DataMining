from pandas import json_normalize
import pandas as pd
import json
import math
import matplotlib.pyplot as plot

jsondata = json.load(open("../../data/plparadigm.json"))
array = []
for data in jsondata:
    array.append([data["year"], data["languageLabel"], data["paradigmLabel"]])
dataframe = pd.DataFrame(array, columns=["year", "languageLabel", "paradigmLabel"])
dataframe = dataframe.astype(
    dtype={"year": "int64", "languageLabel": "<U200", "paradigmLabel": "<U200"}
)
grouped = dataframe.groupby(["paradigmLabel", "year"]).count()
grouped = grouped.rename(columns={"languageLabel": "count"})
grouped = grouped.groupby(["paradigmLabel"])
# Initialization of subplots
nr = math.ceil(grouped.ngroups / 2)
fig, axes = plot.subplots(nrows=nr, ncols=2, figsize=(20, 25))
# Creation of subplots
for i, group in enumerate(grouped.groups.keys()):
    g = grouped.get_group(group).reset_index()
    g.plot(
        x="year", y="count", kind="bar", title=group, ax=axes[math.floor(i / 2), i % 2]
    )
plot.show()

La m√©thode du coude pour d√©terminer le nombre optimal de clusters#

Inertie : Dans le clustering KMeans, l‚Äôinertie (√©galement connue sous le nom de somme des erreurs au carr√©, SSE) est calcul√©e comme la somme des distances au carr√© entre chaque point et son centre de cluster (centro√Øde). L‚Äôid√©e est qu‚Äôen minimisant cette inertie, le mod√®le de clustering trouve des clusters compacts o√π les points sont plus proches de leurs centro√Ødes respectifs.

M√©thode du coude :

  • Lorsque vous tracez l‚Äôinertie en fonction du nombre de clusters (k), vous constatez g√©n√©ralement que l‚Äôinertie diminue √† mesure que k augmente.

  • Initialement, ajouter plus de clusters r√©duit consid√©rablement l‚Äôinertie, car chaque point est affect√© √† un cluster plus sp√©cifique.

  • Apr√®s un certain point, ajouter plus de clusters entra√Æne une r√©duction beaucoup plus lente de l‚Äôinertie, et c‚Äôest l√† que le ‚Äúcoude‚Äù appara√Æt.

  • Le ‚Äúcoude‚Äù repr√©sente le nombre optimal de clusters, car ajouter plus de clusters au-del√† de ce point n‚Äôentra√Æne pas d‚Äôam√©liorations substantielles de la qualit√© du mod√®le (mesur√©e par l‚Äôinertie).

Interpr√©tation du graphique :

  • Sur l‚Äôaxe des x, vous avez le nombre de clusters (k).

  • Sur l‚Äôaxe des y, vous tracez l‚Äôinertie pour chaque valeur de k.

from PIL import Image
import numpy as np
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans

# Charger l'image
imgfile = Image.open("../../images/flower.jpg")
img = np.array(imgfile)

# Redimensionner l'image en un tableau 2D (chaque ligne est un pixel et chaque colonne est un canal de couleur)
pixels = img.reshape(-1, 3)

# Liste pour stocker l'inertie pour diff√©rents nombres de clusters

# Essayer diff√©rentes tailles de clusters de 1 √† 10
for k in range(1, 11):

Quelle est la valeur optimale du nombre de clusters ?

La m√©thode de la silhouette √©value la qualit√© du clustering en mesurant dans quelle mesure chaque point de donn√©es s‚Äôadapte bien √† son cluster assign√© par rapport aux autres clusters. Elle calcule un score de silhouette pour chaque point, d√©fini par :

\(s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}\)

o√π :

  • \(a(i)\) est la distance moyenne entre un point et les autres dans son cluster.

  • \(b(i)\) est la distance moyenne entre le point et le cluster le plus proche.

Le score varie de -1 √† 1 :

  • Proche de 1 ‚Üí Bien regroup√©

  • Autour de 0 ‚Üí Clusters qui se chevauchent

  • Inf√©rieur √† 0 ‚Üí Mauvaise classification

Un score moyen de silhouette plus √©lev√© pour tous les points indique une meilleure configuration de clustering. Cela aide √† d√©terminer le nombre optimal de clusters (K), en compl√©ment de la m√©thode du coude.

from PIL import Image
import numpy as np
import matplotlib.pyplot as plt
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score
from sklearn.utils import shuffle

# Charger l'image
imgfile = Image.open("../../images/flower.jpg")
img = np.array(imgfile)

# Redimensionner l'image en un tableau 2D (chaque ligne est un pixel et chaque colonne est un canal de couleur)
pixels = img.reshape(-1, 3)

# √âchantillonner un sous-ensemble de pixels (par exemple, 10 000 pixels)
sample_size = min(10000, pixels.shape[0])  # Ajuster la taille de l'√©chantillon selon les besoins
pixels_sampled = shuffle(pixels, random_state=42)[:sample_size]

# Listes pour stocker les valeurs d'inertie et les scores de silhouette
inertia_values = []
silhouette_scores = []

# Essayer diff√©rentes tailles de clusters de 2 √† 10 (le score de silhouette n'est pas d√©fini pour k=1)
for k in range(2, 11):
    kmeans = KMeans(n_clusters=k, n_init=10)
    kmeans.fit(pixels)
    
    # Stocker l'inertie
    inertia_values.append(kmeans.inertia_)
    
    # Calculer le score de silhouette uniquement sur les pixels √©chantillonn√©s  
    score = silhouette_score(pixels_sampled, kmeans.predict(pixels_sampled))

    silhouette_scores.append(score)

# Tracer la m√©thode du coude (Inertie)
plt.figure(figsize=(12, 5))

plt.subplot(1, 2, 1)
plt.plot(range(2, 11), inertia_values, marker='o', label="Inertie")
plt.title('M√©thode du coude pour d√©terminer le K optimal')
plt.xlabel('Nombre de clusters (K)')
plt.ylabel('Inertie')
plt.grid(True)

# Tracer les scores de silhouette
plt.subplot(1, 2, 2)
plt.plot(range(2, 11), silhouette_scores, marker='s', color='red', label="Score de silhouette")
plt.title('Score de silhouette pour diff√©rents K')
plt.xlabel('Nombre de clusters (K)')
plt.ylabel('Score de silhouette')
plt.grid(True)

plt.show()
